# Contribution Guidelines

Welcome to **Xaytheon**.
This project is part of **Social Winter of Code (SWoC)** and is open to community contributions.

Please read this document carefully before contributing.

---

## How to Get Started

1. Star the repository (optional but appreciated)
2. Fork the repository
3. Clone your fork locally
4. Create a new branch for your work

   ```bash
   git checkout -b feature/your-feature-name
   ```

---

## Issues and Task Assignment

### Creating Issues

* Contributors are allowed to create their own issues.
* Clearly describe:

  * The problem or feature
  * The proposed solution or improvement
  * Optional references, screenshots, or mockups

### Getting Assigned

* After creating or selecting an issue:

  * Message on the **Discord server** to get the issue assigned.
  * Mention:

    * Issue number
    * What you plan to work on
* Do not start working unless the issue is officially assigned to you.

Pull Requests without issue assignment may be closed.

---

## Discord Communication

* All task discussions and assignments happen on **Discord**.
* Use Discord to:

  * Ask questions
  * Get issues assigned
  * Share progress updates
  * Clarify requirements

Discord communication is mandatory for coordination.

---

## Making Changes

* Follow clean and readable coding practices.
* Keep changes focused (one Pull Request per issue).
* Comment your code where necessary.
* Ensure the project runs without errors.

---

## Submitting a Pull Request

1. Commit your changes with a meaningful message

   ```bash
   git commit -m "Add: short description of feature or fix"
   ```
2. Push your branch

   ```bash
   git push origin feature/your-feature-name
   ```
3. Open a Pull Request to the **main branch**.
4. In the Pull Request description:

   * Mention the issue number (example: `Closes #12`)
   * Explain what was changed
   * Add screenshots if the change is UI-related

---

## Review Process

* Maintainers will review your Pull Request.
* You may be asked to make improvements or fixes.
* Once approved, the Pull Request will be merged.

---

## Contribution Rules

* One issue per contributor at a time
* Be respectful and professional
* Follow GitHub and SWoC guidelines
* No plagiarism
* No spam or low-effort Pull Requests

---

## Beginner-Friendly

* Beginner-friendly issues will be labeled.
* New contributors are welcome.
* Feel free to ask questions on Discord.
